    Using the mouse

    VARKON assumes a mouse with two or three buttons. The left button is always used to make selections and the right button is used to quit the current operation. If a middle button is present it is used to quit the current operation and return to the main menu.

    Many VARKON operations include the use of the mouse to indicate or outline something in a view window. The ZOOM-operation for example where you first press the left button, then move the mouse with the button still down and finally release the button. Entity selection is another example where you use the mouse and its left button or the keyboard to indicate a certain entity. Whenever VARKON wants you to use the mouse in a view window the shape of the cursor changes from the normal pointer to a crosshair.

    VARKON also supports the use of the keyboard together with the mouse. Here is a complete list of keys that can be used instead of the mouse buttons. Each key is actually configured through a t-string in $VARKON_MDF/texts.INC.

    Key t-string Description
    p t80 Nearest point
    l t81 Nearest line
    a t82 Nearest arc
    c t83 Nearest curve
    s t84 Nearest surface
    C t85 Nearest coordinate system
    t t86 Nearest text
    L t87 Nearest linear dimension
    D t88 Nearest diameter dimension
    R t89 Nearest radius dimension
    A t90 Nearest angular dimension
    X t91 Nearest xhatch
    c t92 Same as left mousebutton
    g t93 Nearest position in active grid
    P t95 Nearest part
    b t96 Nearest b_plane
    i t97 Enter identity from keyboard
    w t98 Use window mode
    SPACE t99 Same as left mousebutton
    o t102 Outside window
    c t103 Completely inside/outside
    RETURN - Quit the operation
    TAB - Quit and return to main menu
    ctrl-a - Display help
    ctrl-b - Show position-menu
