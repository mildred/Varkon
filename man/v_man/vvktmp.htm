    VARKON_TMP

    Environment variable


    Description

    Environment variable used by VARKON to place temporary files. Files created in $VARKON_TMP are normally removed automatically when no longer needed. Should any files be found in $VARKON_TMP when VARKON is not running it is suggested you remove them.


    Comments

    If VARKON_TMP is not defined a hard coded default will be used.

    If VARKON_TMP points to a non existent directory or if read and write permissions to $VARKON_TMP are missing the result is unpredictable.

    It can be a good idea to use /tmp (UNIX) or C:\TEMP (Windows95) for VARKON_TMP since these directories are usually on the local macine (not on a networked fileserver) and are also used by other programs. Doing this can produce less trafic on a network and make it easier to perform periodic clean up of unused files.
