pca - Function

Description

Low level routine to create a 4x4 matrix by using a statistical method called Principal Component Analysis(PCA) and specifying set of points "point cloud" stored into a one dimension matrix. The result is a 4x4 matrix, which might be represents a coordinate system or transformation matrix.

For more details see "Mathematics for 3D Game Programming and Computer Graphics" Authored by Eric Lengyel - second edition, pages 217--221.

Syntax

status:=pca(npts,pts,tmat);
INTEGER status;
INTEGER npts;
VECTOR pts(npts);
FLOAT tmat(4,4);

Principal parameters

npts - Number of points
pts - Array of points
tmat - 4x4 transformation matrix

Optional parameters

None.

Return Value

Returns 0 if the 4x4 transformation matrix is created and -1 if the 4x4 is not created.

Comments

The created 4x4 matrix will be like the following matrix:

Xx Xy Xz Ox
Yx Yy Yz Oy
Zx Zy Zz OZ
0 0 0 1

Since:

Xx, Xy, Xz => The unit length eigenvector components correspond to the largest eigenvalue.
Yx, Yy, Yz => The unit length eigenvector components correspond to the second largest eigen value.
Zx, Zy, Zz => The unit length eigenvector components correspond to the smallest eigen value.
Ox, Oy, Oz => The mean of set of points represents the translation transformation.

The three eigen vectors are orthogonal to each other.

To visualize or use 4x4 matrix that represent the coordinate system use csys_usdef() procedure see below.

See also

csys_usrdef() and tform_usrdef
