Ini-files

Ini-files are used by Varkon to initialize GUI related parameters and values during startup (in the X-windows community such things are called "resources"). The standard Varkon distribution includes a default ini-file but you can also create a personal ini-file with settings that will override the default values. With the ini-file you can control things like:

  1. Text fonts used in menus and dialogs
  2. Which text editor to use for MBS programming
  3. Which browser to use for docs and help
  4. Size, type and position of default graphic windows
  5. Toolbars in graphic windows
  6. Texts and tooltips for all dialogs
  7. Colors used for dialogs, buttons, shadows etc
  8. Default colors for Varkon pens
  9. Settings for OpenGL

See the comments in the ini-file for a complete description of the resources defined and how they influence the behavior of the user interface.

Personal ini files

Each time you start Varkon, Varkon will load the default ini-file. The default ini-file is specified in the Varkon start script. See start script. It is also possible to instruct Varkon to load a second ini-file. A setting in the second file will override the same setting in the first file.

You can use this mechanism to create your own personal ini-file. To avoid that your personal ini-file will be lost if you update your Varkon system to a newer version you should keep your personal ini-file in a location on your hard disc outside the Varkon root directory.

To configure Varkon to use your personal ini-file, edit the line that starts the Varkon executable according to the instructions given in the default start script.
